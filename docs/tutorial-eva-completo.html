Como usar a EVA Agent por completo

Guia prático para Windows: instalar, conversar no terminal, deixar o Desk na bandeja, voz, Telegram e o que cada peça faz. Sem enrolação.

Onde está o software
  • Core (agente de verdade): C:\Users\mathe\Documents\eva-agent · GitHub Pasqualotty/eva-agent
  • Desk (bandeja / voz / UI): C:\Users\mathe\Documents\evagent · GitHub Pasqualotty/evagent
  • Dados e secrets: %LOCALAPPDATA%\eva (config, .env, logs, sessões)

1. O que é a EVA

A EVA é o seu agente pessoal: conversa, usa tools (terminal, arquivos, web, browser…), tem skills, memória entre sessões, cron e gateway de mensagens. O motor é um fork completo do Hermes Agent (Nous Research), rebrandizado e configurado pra você.

EVA Agent (core)

CLI eva: cérebro + tools + skills + gateway Telegram/Discord/etc. É o “agente de verdade”.

EVA Desk

App Windows: bandeja, hotkey PTT, voz Eva Humana (Voicebox), painel de chat. Pode falar com o core via /ping e brain eva-core.

3. Requisitos

  • Windows 10/11
  • Python 3.11–3.13 (recomendado 3.11)
  • Git
  • Chave de modelo (xAI/Grok, OpenAI, Anthropic, OpenRouter, etc.) — via eva setup ou .env
  • Opcional Desk: Voicebox instalado + perfil Eva Humana
  • Opcional Telegram: bot criado no @BotFather

4. Instalar o core

4.1 Caminho rápido (desenvolvimento)

1
Abra PowerShell e rode: 
cd C:\Users\mathe\Documents\eva-agent
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\bootstrap_dev.ps1
Isso cria .venv, instala o pacote editável e testa eva --help.
2
Ative o ambiente e entre no agente: 
.\.venv\Scripts\Activate.ps1
eva --help
eva setup
eva
Sem ativar o venv: 
.\.venv\Scripts\eva.exe
.\.venv\Scripts\python.exe -m hermes_cli.main --help

4.2 PATH permanente (recomendado no dia a dia)

Para digitar eva em qualquer terminal novo:

cd C:\Users\mathe\Documents\eva-agent
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\install-user-path.ps1
Atenção: feche e abra o terminal (ou faça logoff) para o PATH novo valer. Use -DryRun se quiser só simular.

Documentação detalhada: docs\install-windows.md.

4.3 Configurar o modelo (API)

  1. Rode eva setup e escolha o provider (xAI/Grok, OpenAI, Anthropic, OpenRouter…).
  2. Ou edite manualmente %LOCALAPPDATA%\eva\.env (nunca commite este arquivo).
  3. Teste: eva e mande “oi”.
# Exemplo de chaves (use só as que você tiver)
# %LOCALAPPDATA%\eva\.env
EVA_HOME=C:\Users\mathe\AppData\Local\eva
XAI_API_KEY=...
OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
TELEGRAM_BOT_TOKEN=...   # só se for usar gateway

5. Usar no terminal (core)

O modo principal de “agente completo” é a CLI interativa. É aqui que skills, tools, memória e multi-passo brilham.

5.1 Comandos essenciais

ComandoO que faz
evaAbre chat interativo (TUI/CLI)
eva setupWizard de provider, modelo e basics
eva modelTroca/lista modelos
eva doctorDiagnóstico do ambiente
eva toolsQuais toolsets estão ativos
eva skillsLista / gerencia skills
eva gateway …run / status / install do messaging
eva deskStatus da ponte desktop (stub/info)
eva -z "texto"One-shot: pergunta e resposta sem sessão interativa
eva updateAtualiza instalação gerenciada (se aplicável)

5.2 Dentro do chat (slash)

  • /new ou /reset — conversa nova
  • /model … — troca modelo
  • /skills — skills disponíveis
  • /compress — comprime contexto longo
  • /usage — uso de tokens
  • /stop — interrompe trabalho em andamento (no gateway também)

Muitos outros slash commands vêm de skills instaladas (aparecem como /nome-da-skill).

5.3 Skills e memória

  • Skills = procedimentos reutilizáveis (SKILL.md). O agente pode criar/atualizar skills com a experiência.
  • Memória = fatos curtos em MEMORY/USER + busca em sessões antigas (FTS5).
  • Pasta típica: sob %LOCALAPPDATA%\eva (memories, sessions, skills do perfil).
Dica: depois de um workflow chato que deu certo, peça explicitamente: “Salva isso como skill para a próxima vez.”

5.4 Tools e permissões

O agente pode executar terminal, ler/escrever arquivos, buscar na web, browser, etc. Comandos perigosos pedem aprovação (a menos que você rode em modo bem permissivo).

  • Revise toolsets: eva tools
  • Para dev pesado em repositório, prefira combinar com o Grok TUI / pipeline EVA no cwd do projeto — o core não substitui o maestro multi-frente.

6. EVA Desk (desktop Windows)

O Desk é a face “sempre no ar”: ícone na bandeja, hotkey de voz, painel de chat. Ele não é um segundo Hermes — é a casca Windows que pode chamar o core.

6.1 Subir o residente

1
Duplo clique em:
C:\Users\mathe\Documents\evagent\Iniciar EVA Agent.bat
(ou scripts\launch_eva_agent.ps1 / python -m eva_desk --resident)
2
Procure o ícone EVA na bandeja do Windows. Menu típico: Abrir painel · Ping core · Versão · Mutar · Sair.
3
Tooltip do tray deve mostrar algo como EVA Agent · core ok se o CLI eva estiver no PATH/venv. Se estiver core offline, o core não foi encontrado — rode o bootstrap/PATH.

6.2 Voz e PTT

  • PTT (hold-to-talk): padrão Ctrl+Shift+Space (confirme no painel com /ptt se existir).
  • Segure → fale → solte → STT (Voicebox) → texto no chat → resposta → TTS Eva Humana.
  • Voicebox precisa estar instalado e saudável (API local, perfil Eva Humana).
  • Playback é interno — não deve abrir o Media Player do Windows.

6.3 Cérebros no Desk e chat com o core

Comando / configEfeito
/brain grokDefault: Grok via ACP (tools de código no cwd)
/brain openai / claudeChat HTTP (precisa de API key no ambiente)
/brain eva-coreOne-shot via CLI eva (subprocess)
/pingTesta se o binário eva responde
/modo seguro|obra|sistemaNíveis L1/L2/L3 de permissão de ações
/briefResumo de contexto (projeto, git, janela)
/helpLista de comandos do Desk
Importante: no Desk, eva-core é caminho de chat/ping. O “agente Hermes completo” (tools ricas, skills, multi-step) continua sendo a CLI eva ou o Telegram via gateway.

7. Telegram (gateway)

Com o gateway ligado, você fala com a EVA de qualquer lugar — o mesmo agente do core, com tools e sessão.

7.1 Ligar o gateway

1
Crie o bot no @BotFather e copie o token.
2
Coloque no arquivo %LOCALAPPDATA%\eva\.env: 
TELEGRAM_BOT_TOKEN=123456:ABC-DEF...
Nunca commite esse arquivo no Git.
3
Instale a lib do Telegram no venv (se o doctor reclamar): 
cd C:\Users\mathe\Documents\eva-agent
.\.venv\Scripts\python.exe -m pip install "python-telegram-bot>=21"
4
Suba o gateway: 
powershell -NoProfile -ExecutionPolicy Bypass -File C:\Users\mathe\Documents\eva-agent\scripts\start-gateway.ps1
Parar: 
powershell -NoProfile -ExecutionPolicy Bypass -File C:\Users\mathe\Documents\eva-agent\scripts\stop-gateway.ps1
5
Confira o log (prova real de que está vivo): 
Get-Content "$env:LOCALAPPDATA\eva\logs\gateway.log" -Tail 40 -Wait
Procure linhas como telegram connected e Gateway running.
eva gateway status às vezes diz “not running” mesmo com o processo no ar. Confie no log + processos se o bot estiver respondendo.

7.2 Primeira mensagem e segurança

  1. Abra o bot no Telegram (ex.: o username que o getMe mostrou).
  2. Mande “oi”.
  3. Se pedir pairing / aprovação, use os comandos de pairing da CLI (eva pairing … (list / approve) conforme a mensagem do bot.
  4. Opcional: restrinja usuários com TELEGRAM_ALLOWED_USERS no .env (IDs numéricos). Ver docs\gateway.env.example e docs\gateway-windows.md.
Segurança: se o token vazar em chat/screenshot, revogue no BotFather (/revoke) e atualize o .env. Trate o token como senha.

8. Dia a dia

8.1 Fluxos recomendados

SituaçãoUse
Pergunta rápida no celular Telegram (gateway ligado no PC)
Tarefa com tools, código, multi-passo CLI eva no projeto certo (cd antes)
Mãos no teclado + voz no PC Desk + PTT; brain Grok ou eva-core
Engenharia multi-frente / PRs / pipeline Grok TUI + maestro EVA (não o gateway)
Automação diária Cron do core (eva cron / skills de schedule)

8.2 Atualizar o core

cd C:\Users\mathe\Documents\eva-agent
git pull origin main
.\.venv\Scripts\Activate.ps1
pip install -e ".[cli]"
# se mudou deps de Telegram:
pip install "python-telegram-bot>=21"
# reinicie o gateway se estiver rodando

8.3 Onde olhar quando algo quebra

  • Logs core/gateway: %LOCALAPPDATA%\eva\logs\
  • Config Desk: %APPDATA%\EvaDesk\config.json (nome legado da pasta)
  • Docs no repo: docs\install-windows.md, docs\gateway-windows.md, docs\DESKTOP-SHELL.md, docs\EVA-PARITY.md

9. Problemas comuns

SintomaO que fazer
eva não é reconhecido Ative o venv ou rode install-user-path.ps1 e abra terminal novo
Tray “core offline” Instale/PATH do eva; menu Ping; confira where.exe eva
Sem voz no Desk Voicebox aberto/saudável; perfil Eva Humana; /voice no Desk
Gateway status “not running” Olhe gateway.log e Task Manager por eva.exe / python gateway
Telegram não conecta Token no .env; pip python-telegram-bot; firewall; log
Bot não responde à sua msg Pairing / allowlist de user IDs; reinicie gateway
Modelo recusa / 401 eva setup / keys no .env; eva doctor
